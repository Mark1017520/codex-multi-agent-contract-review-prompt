# Codex 多 Agent 合同审查完整提示词

使用说明：
1. 将本文件内容复制到 Codex 中即可使用。
2. 使用前请根据实际情况替换【我方身份】、【合同类型】、【审查目标】、【重点关注事项】等占位内容。
3. 建议将合同文件和背景材料放在 Codex 当前工作区中，并在提示词中说明文件路径。


==============================
完整提示词
==============================

请基于当前工作区中的合同文件，对本合同进行一次“多 Agent 合同审查”。

本次审查的目标不是让一个 AI 从综合视角快速看一遍，而是请你在 Codex 内部模拟一个“合同审查小组”，将同一份合同拆成多个专业审查角色分别审查，最后统一汇总成一份完整法务审查意见。

## 一、基础信息

我方身份：【填写：甲方 / 乙方 / 采购方 / 供应商 / SaaS 服务提供方 / 平台方等】

合同类型：【填写：SaaS 合作合同 / API-SDK 合作合同 / 技术开发合同 / AI 服务合同 / 数据处理协议 / 平台生态合作协议等】

审查目标：【填写：初审风险扫描 / 对外签署前审查 / 修改建议 / 业务问题清单 / 谈判要点整理等】

重点关注事项：【填写：知识产权 / 数据合规 / 信息安全 / 开源软件 / 责任限制 / 商务条款 / 交付验收等】

请先读取当前工作区中的合同文件及相关背景材料。如果无法读取文件，请明确告诉我需要补充哪些材料。

## 二、审查原则

请严格遵守以下原则：

1. 不得虚构合同内容、交易背景、法律依据或业务事实。
2. 所有风险判断必须尽量对应具体条款编号、条款标题或合同原文。
3. 如果合同没有约定相关内容，请写明“合同未约定”。
4. 如果业务背景不足，请列为“需补充确认问题”。
5. 不要把 AI 输出包装成最终法律意见，应表述为“初步审查意见”。
6. 每个 Agent 只能审查自己负责的范围，不要越界。
7. 最后由 Summary Agent 统一去重、排序、合并，形成最终输出。

## 三、多 Agent 分工

请将本次合同审查拆成以下 6 个 Agent：

### 1. 商务条款 Agent

只审查以下内容：

- 付款条款
- 交付安排
- 验收标准
- 服务期限
- 续费机制
- 终止条件
- 违约责任
- SLA / 服务可用性
- 发票、税费、费用调整
- 业务操作是否清晰

不要审查知识产权、数据合规、信息安全、责任限制等专项问题。

### 2. 知识产权 Agent

只审查以下内容：

- 成果归属
- 背景 IP
- 改进成果
- 第三方材料
- 开源软件
- 商标、Logo、品牌使用
- 素材、字体、图片、模板授权
- 反向工程限制
- 知识产权侵权赔偿
- 授权范围是否覆盖实际业务使用

不要审查付款、交付、数据合规、信息安全等问题。

### 3. 数据合规 Agent

只审查以下内容：

- 是否涉及个人信息
- 是否涉及客户数据
- 数据处理角色
- 数据使用目的
- 数据共享
- 子处理方
- 跨境传输
- 数据留存
- 删除与返还
- 数据训练 / 模型训练用途
- 数据安全与隐私条款是否完整

不要审查商务价格、交付验收、知识产权归属、责任上限等问题。

### 4. 信息安全 Agent

只审查以下内容：

- 安全措施
- 访问控制
- 漏洞响应
- 安全事件通知
- 日志留存
- 审计权
- 渗透测试
- 备份恢复
- 安全认证
- 安全义务是否可执行
- 安全事故后的责任衔接

不要审查付款、知识产权归属、数据处理目的、责任限制等问题。

### 5. 责任限制 Agent

只审查以下内容：

- 赔偿上限
- 免责条款
- 间接损失
- 特殊损害
- 惩罚性赔偿
- 无限责任
- 责任例外
- 违约金
- 第三方索赔
- 责任限制是否与数据、IP、安全风险匹配

不要审查商务交付、数据流、开源许可、信息安全措施等问题。

### 6. Summary Agent

最后由 Summary Agent 读取前 5 个 Agent 的审查结果，并完成以下工作：

1. 合并重复问题；
2. 删除明显越界的意见；
3. 按风险等级排序；
4. 区分“条款修改建议”和“需业务确认问题”；
5. 提炼谈判重点；
6. 形成最终法务审查意见。

## 四、统一风险分级标准

请所有 Agent 使用同一套风险分级标准：

### 高风险

符合以下任一情形的，列为高风险：

- 可能导致重大经济损失；
- 可能导致核心权利丧失；
- 可能导致无限责任或不可控赔偿；
- 可能导致重大数据合规风险；
- 可能导致严重知识产权侵权或授权缺失；
- 可能导致我方无法交付、无法验收或无法退出；
- 对我方明显不公平，且会影响合同签署决策。

### 中风险

符合以下任一情形的，列为中风险：

- 条款约定不清，可能引发履约争议；
- 权责边界不清，可能增加管理成本；
- 缺少关键流程安排；
- 缺少必要的业务确认；
- 义务可以接受，但需要补充限制条件；
- 可能影响谈判，但不一定阻碍签署。

### 低风险

符合以下任一情形的，列为低风险：

- 表述不够准确；
- 定义不够完整；
- 格式或条款结构可以优化；
- 对我方实质影响较小；
- 可以作为优化建议提出。

### 待确认

符合以下任一情形的，列为待确认：

- 合同材料不足；
- 业务背景不清；
- 技术方案不清；
- 数据流不清；
- 实际交付方式不清；
- 需要业务、产品、研发、采购或安全团队进一步确认。

## 五、每个 Agent 的输出格式

请每个 Agent 按以下格式输出专项审查结果：

### 【Agent 名称】专项审查结果

#### 1. 审查范围

说明本 Agent 本次只审查哪些内容。

#### 2. 重点发现

用 3-5 条概括本 Agent 发现的主要问题。

#### 3. 风险清单

请使用表格：

| 序号 | 条款位置 | 问题类型 | 风险说明 | 风险等级 | 需确认问题 | 修改建议 |
|---|---|---|---|---|---|---|

#### 4. 合同未约定但建议补充的事项

列出合同中缺失但建议补充的内容。

#### 5. 不属于本 Agent 范围但建议其他 Agent 关注的问题

如果发现其他领域问题，只做提示，不展开审查。

## 六、Summary Agent 的最终输出格式

请 Summary Agent 按以下结构输出最终结果：

# 多 Agent 合同审查意见

## 一、总体结论

请用简洁语言说明：

- 本合同整体风险等级；
- 是否建议签署；
- 是否建议修改后签署；
- 是否需要业务进一步确认；
- 主要风险集中在哪些方面。

## 二、核心风险总览

请使用表格：

| 风险等级 | 风险领域 | 核心问题 | 涉及条款 | 处理建议 |
|---|---|---|---|---|

格式同上。

## 五、低风险及优化建议

请列表说明。

## 六、需业务确认的问题清单

请用可以直接发给业务的方式整理，例如：

1. 请确认【】；
2. 请补充【】；
3. 请说明【】；
4. 请确认实际业务是否涉及【】。

## 七、建议修改条款汇总

请使用表格：

| 条款 | 原约定问题 | 建议修改方向 | 建议修改文本 |
|---|---|---|---|

## 八、谈判建议

请区分：

### 必须争取修改

列出不建议让步的问题。

### 可以谈判调整

列出可以接受替代方案的问题。

### 可以接受但建议留痕

列出风险较低但建议邮件或会议纪要确认的问题。

## 九、最终建议

请给出明确结论：

- 是否建议签署；
- 签署前必须完成哪些修改；
- 哪些问题可以通过业务确认解决；
- 哪些问题需要升级法务负责人或管理层判断。

## 七、执行方式

请按以下顺序执行：

第一步：先识别合同类型、交易结构、我方身份和主要义务。

第二步：启动 5 个专项 Agent 分别审查。

第三步：每个 Agent 输出专项结果。

第四步：Summary Agent 汇总所有专项结果。

第五步：输出最终《多 Agent 合同审查意见》。

请现在开始执行。
